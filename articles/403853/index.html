Log and database IoT events. Quick start from iRidium server

Quite often it is necessary to see when and what happened in the automation system. For serious projects, you need to have access to a database of such events, often for a long time.



An accident happened, the door opened, the sensor was triggered. The temperature dropped to -10, so the pipe burst. Someone came to the office at night - camera recordings were lost, but here are the logs of motion sensors. In general, a useful thing.

On the basis of iRidium, such bases are made quite simply, and locally, without using clouds or SCADA servers. Just add the tags of the protocols you need, and then mark them for saving to the database.



By default, databases are saved to the "_user directory_ \ Documents \ iRidium pro documents \ Server \ Database" folder.

At the same time, there is a main base in which you need to understand a little, but there is a custom one that records only the data we need in the right format. Here is the database API , if someone wants to understand in detail.

Pleasant moment: if you use up to 50 tags, the server license for the integrator is completely free.
