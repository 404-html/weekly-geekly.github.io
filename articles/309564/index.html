Centralized backup policy management

Another good news from the Microsoft Azure development team. If any of the readers who already use the backup service in the Azure ARM portal, they may have noticed that managing backup policies is extremely difficult. Indeed, you were given the opportunity to select an existing or create a new policy, only when adding a new server to the Recovery Services Vault .


Now everything has changed. Literally today , the Microsoft Azure team added the ability to work with backup policies in a separate snap-in of the Azure Resource Manager portal.

Those who tried to configure backup policies in Recovery Services Vaul t, remember that this could be done only when adding a new server.


The most inconvenient thing about this was that you couldn‚Äôt see all the available backup policies directly on the portal in a convenient way. It is clear that this could be done through PowerShell, but it was all the same strange that the portal lacks this functionality.

Now everything is wrong. Starting today, you have the opportunity to open the Backup Policies snap-in right in the properties of the Recovery Services Vault and see all your policies.


Here you can, respectively, change the settings of existing policies or add new ones.

In conjunction with the recent introduction of monitoring functionality and alerts for Azure Backup , you can only rejoice at the speed with which this service is developing.
